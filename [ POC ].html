[ POC ] # POC : Proof Of Concentpt # VOC : Voice Of Customer wireframe 수준의 인사이트 ui gui RFP 분석 - tworld 프로젝트 설명 [배경 및 목적] 단절돼 있는 마케팅과 UX를 통합하고 흩어져 있는 POC를 통합해 UX 관점의 POC 통합 마케팅 플랫폼을 구축했다. 이로써, 유무선 서비스를 T world 중심으로 통합해 상품 및 디바이스 판매를 허브화하고 오프라인 유통망의 한계를 극복해 온라인과 O2O 유통채널의 성장을 견인하고자 한다. [콘셉트] 첫째, 끌고 가는 힘으로서 통합 메인 UI 및 UX를 구축했다. 둘째, 데이터를 통합해 연동하는 통합을 실현하고자 했다. 각 POC 간의 서버 데이터를 하나의 기능 공간 안에서 연동돼 구성하도록 하기 위함이다. 둘째, 연합하는 통합으로서 UI를 통합했다. 각 POC 별 동일 카테고리 콘텐츠를 하나의 전시공간 안에 모아 구성했다. 셋째, 연계하는 통합으로서의 가이드 통합을 실현했다. 세부 처리 단계에 접속 시 해당 POC로 이질감 없이 자연스럽게 환승시켜 준다. [주요 기술] 첫째, T-ABLE : T라서 가능한 것들이다. 이는 SKT가 고객에게 전달하고자 하는 가치다. 주요 기능으로는 통합 회선 정보로 회선을 변경해 정보를 확인하는 번거로움을 개선했다. 또한, 로그인한 고객이 보유하고 있는 모든 회선 정보를 통합해 조회 가능하다. 회선정보 변경 공통 기능을 사용해 어떤 페이지든지 원하는 회선 정보로 변경해 해당 회선에 맞는 정보 확인이 가능하다. 둘째, T-EASER : T라서 쉬워진 것들이다. 이는 개별 POC가 통합 티월드에 기대하는 가치다. 주요 기능으로는 먼저 알파내비게이션 제공으로 고객 맞춤형 서비스를 통해 고객이 원하는 정보를 쉽게 탐색할 수 있도록 구성했다. 또한, 고객정보 보호를 위해 가려진 정보는 2차 인증 기능을 제공함으로써 개인 정보 보호 부분을 고려했다. 셋째, T-WIN : T라서 얻어갈수 있는 것들이다. 이는 고객이 SKT로부터 받고 싶은 가치다. 주요 기능으로는 상품별 최적화된 테마다. 대화형으로 고객이 원하는 정보를 쉽게 찾아갈 수 있도록 구성했다. 고객 유형, 상황별로 내게 맞는 최적의 상품 서비스 찾기 기능을 제공해 방대한 상품 서비스를 직접 찾고 비교하지 않아도 원하는 상품을 손쉽게 찾을 수 있도록 구성했다.
